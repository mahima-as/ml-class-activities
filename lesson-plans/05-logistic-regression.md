## Learning Objectives

By the end of this activity, students will be able to:

1. Apply logistic regression using scikit-learn to a binary classification problem.
2. Interpret a confusion matrix and explain what each cell represents.
3. Compute and interpret accuracy, precision, and F1 score.
4. Explain the difference between precision and recall and when each matters more.

---

## Instructor Notes

### Setup
- This notebook uses a pre-computed dataset (`separable2vs8.csv`) extracted from
  ZipDigits. The dataset contains intensity and symmetry features for digits 2
  and 8 that are linearly separable.
- The notebook uses `sklearn.linear_model.LogisticRegression` — no implementation
  from scratch is required. The focus is on evaluation, not training.

### Introducing the activity (3 min)
Start with a question rather than an explanation:

> "If I told you my classifier is 90% accurate, would you trust it? What else
> would you want to know?"

Let students respond. Use their answers to motivate precision, recall, and the
confusion matrix before they run any code.

### Common Issues
- Students often confuse precision with accuracy. A brief verbal example (e.g.,
  a spam filter that marks everything as spam) can clarify.
- The `precision_score` and `f1_score` functions in sklearn require the
  `pos_label` argument if the labels are not 0/1. With labels {−1, +1}, students
  may need to specify `pos_label=1`.
- If the dataset is not available, students can quickly generate it by running
  the first half of `Digits Linear Models.ipynb`.

### Facilitation Tips
- Ask students to interpret the confusion matrix entry by entry before moving
  to computed metrics. "What does this number mean in plain English?"
- If time allows, ask: "What would you change about the model if false negatives
  are much more costly than false positives?" (e.g., medical diagnosis context).

---

## Discussion Prompts

1. What does a high precision but low recall tell you about a classifier?
   Give a real-world example where this matters.
2. Your model achieves high accuracy on this dataset. Can you trust that number?
   What could make accuracy misleading?
3. The dataset here is linearly separable — the logistic regression fits almost
   perfectly. How do you expect performance to change on a harder dataset?
4. Logistic regression outputs probabilities. What threshold did sklearn use by
   default? What would happen if you changed it?

## Pass/Fail Specifications

To **pass** this activity, a submission must:

- [ ] Successfully load the dataset and fit a logistic regression model
- [ ] Display the confusion matrix and correctly label its four cells
      (TP, FP, FN, TN) in a comment or markdown cell
- [ ] Report accuracy, precision, and F1 score
- [ ] Include a 2–3 sentence interpretation of the results: what do these
      metrics say about the model's performance?

Specifications will be confirmed and released when the activity is discussed in class.
Students may revise and resubmit up to two times.
